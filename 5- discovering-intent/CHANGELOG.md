# Changelog

All notable changes to the `discovering-intent` skill will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.1.0] - 2026-01-08

### Changed
- **Renamed** from `interview` to `discovering-intent` (clearer purpose)
- **Renamed** wrapper folder from `5- interview` to `5- discovering-intent`
- Updated description to remove slash command reference (skills auto-trigger)

## [1.0.0] - 2026-01-06

### Added
- Initial release
- WHY + WHAT discovery model for understanding user intent
- Techniques for uncovering intent:
  - Laddering (dig into abstract goals)
  - 5 Whys (uncover root need)
  - Surface assumptions explicitly
- Structured clarification patterns (Required vs Optional)
- Agreement confirmation before proceeding
- Reference documentation:
  - `question-patterns.md` - Techniques for discovering WHY
  - `anti-patterns.md` - Common mistakes to avoid
  - `summary-templates.md` - Output formats for different situations
- Depth check criteria for sufficient understanding
- Scenario-based formality (Quick, Standard, Detailed outputs)
